## 老师先讲：四种最常用数据结构

### list

`list` 是有顺序、可修改的序列。

常见场景：

- 保存一批样本
- 保存一组分数
- 按顺序处理数据

示例：

```python
scores = [85, 92, 78]
names = ["hka", "tom", "alice"]
```

特点：

- 有顺序
- 可重复
- 可以修改

### dict

`dict` 是“键值对”结构。

常见场景：

- 一条样本的字段信息
- 参数配置
- 统计词频、标签计数

示例：

```python
student = {
    "name": "hka",
    "score": 95,
    "city": "Hangzhou"
}
```

特点：

- 通过 key 查 value
- 适合表示结构化信息
- 是以后处理 JSON 的核心

### set

`set` 是无序且元素唯一的集合。

常见场景：

- 去重
- 判断某元素是否存在

示例：

```python
tags = {"python", "ai", "python", "ml"}
```

结果里重复项会自动消失。

特点：

- 自动去重
- 适合做成员判断
- 不保留重复值

### tuple

`tuple` 是有顺序、但不可修改的序列。

常见场景：

- 不希望被改动的一组固定值
- 函数返回多个值

示例：

```python
point = (3, 5)
```

特点：

- 有顺序
- 不可修改
- 适合固定结构

## 你先建立一个判断标准

以后看到题目或场景，可以这样判断：

1. 我需不需要顺序？
2. 我需不需要通过名字查内容？
3. 我需不需要去重？
4. 这份数据后面还会不会改？

如果你能用这四个问题判断，大多数基础数据结构选择都不会错。

## 跟做环节 1：先写一个最小例子

在 Day-002 的 `code/` 文件夹里创建 `data_structures_demo.py`，写入：

```python

```

运行后观察四种结构的输出长什么样。

## 跟做环节 2：练 4 个基础操作

继续在同一个文件里增加：

```python
names.append("bob")
student["city"] = "Shanghai"
tags.add("llm")

print(names[0])
print(student["score"])
print("ai" in tags)
print(point[1])
```

你要看懂的是：

- `append` 是给 list 追加元素
- `dict[key]` 可以取值或赋值
- `set.add()` 可以新增元素
- `in` 常用于判断元素是否存在
