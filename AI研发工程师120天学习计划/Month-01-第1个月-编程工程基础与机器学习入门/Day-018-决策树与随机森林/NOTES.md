## 核心笔记

- 决策树更像在不断问问题、按条件一步步分裂做判断。
- 随机森林是很多棵决策树一起投票的集成模型。
- 决策树是单模型，随机森林是多模型集成，这是它们最核心的直观区别。
- 单棵决策树更容易走一条很果断的分裂路径，甚至可能过度依赖某一个特征。
- 随机森林由于由多棵树组成，每棵树看到的数据和特征子集不完全一样，所以通常更稳、更不容易过拟合。
- `feature_importances_` 可以帮助观察模型认为哪些特征更重要。
- 如果特征重要性是 `[1.0, 0.0]`，说明模型当前几乎完全依赖第一个特征做判断。

## 代码/实验结果

- 代码核心输出：

```text
tree predictions = [0 0 0 1 1 1]
forest predictions = [0 0 0 1 1 1]
tree feature importance = [1. 0.]
forest feature importance = [0.72 0.28]
```

- 结果理解：
  - 两个模型都把训练数据学对了。
  - 决策树的特征重要性 `[1. 0.]` 表明它几乎完全依赖第一个特征。
  - 随机森林的特征重要性 `[0.72 0.28]` 说明它也更看重第一个特征，但同时利用了第二个特征。

- 结果说明：
  随机森林比单棵树更像“多位评委综合投票”，因此通常更稳。

## 遇到的问题

| 问题 | 原因 | 解决方式 |
| --- | --- | --- |
| 一开始把“随机森林更稳”简单理解成“用了更多特征” | 还没有把重点放在集成思想上 | 修正为：更稳的关键是多棵树集成，降低单棵树的偶然性和过拟合风险 |
