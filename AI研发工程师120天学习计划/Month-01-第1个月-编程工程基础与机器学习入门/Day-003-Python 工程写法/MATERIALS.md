# Day 3 学习资料：Python 工程写法

## 你今天要学会什么

完成一个可复用 Python 脚本

## 核心概念

- 复盘不是写流水账，而是把输入、行动、结果和下一步改进串起来。
- AI 研发岗位很看重可解释的项目经历：你为什么这样建模、指标如何变化、失败样例说明了什么。
- 简历和面试材料要围绕证据组织：代码、实验表、截图、评测集、badcase、部署方式。

## 学习步骤

1. 先回看本阶段所有 README 中的今日目标，把未完成项标记出来。
2. 把代码、笔记、图表、错误记录整理成一个可以被别人读懂的目录。
3. 用 STAR 方法写项目：场景 Situation、任务 Task、行动 Action、结果 Result。
