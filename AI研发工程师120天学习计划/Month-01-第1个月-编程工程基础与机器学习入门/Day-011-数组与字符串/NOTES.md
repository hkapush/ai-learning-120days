## 核心笔记

- 数组和字符串很多题，本质上是在考能否用更少的重复遍历解决问题。
- 当数组是有序的，并且题目和“两端关系”有关时，可以优先考虑双指针。
- 双指针的典型思路是：左右指针从两端向中间收缩，根据当前状态决定移动哪一边。
- 滑动窗口适合处理连续区间类问题，尤其是“最长/最短子串、子数组”这类题。
- 在滑动窗口中，右指针通常负责扩张窗口，左指针通常负责收缩窗口，两者共同维护一个合法区间。
- 处理窗口不合法的情况时，`while` 比单次 `if` 更稳，因为它能在当前轮次内持续把窗口修正到合法状态。
- 这不是说 `if` 一定完全错，而是 `while` 更标准、更直接、更不容易在边界条件下出问题。

## 代码/实验结果

- 双指针题代码：

```python
def find(nums,target):
    left = 0
    right = len(nums) - 1
    while left < right:
        current_sum =  nums[left]+nums[right]
        if(current_sum ==target):
            return [left,right]
        elif current_sum < target:
            left +=1
        else:
            right -=1

    return []

nums = [1, 2, 4, 6, 10]
target = 8

print(find(nums, target))
```

- 输出结果：

```text
[1, 3]
```

- 这题的关键结论：
  有序数组中的两数之和问题，可以把暴力 `O(n^2)` 优化到双指针 `O(n)`。

- 滑动窗口题代码：

```python
def find_2(nums):
    left = 0
    right = 0
    inset =set()
    max_length =0
    while right< len(nums)-1:
        if(nums[right] in inset):
            inset.remove(nums[left])
            left +=1
        else:
            inset.add(nums[right])
            right +=1

        max_length = max(max_length,len(inset))

    return max_length

    
s = "abcabcbb"
print(find_2(s))
```

- 输出结果：

```text
3
```

- 结果说明：
  当前实现对示例是正确的，但标准滑动窗口写法会更倾向使用 `while` 持续收缩窗口，保证每一轮都恢复到合法状态。
