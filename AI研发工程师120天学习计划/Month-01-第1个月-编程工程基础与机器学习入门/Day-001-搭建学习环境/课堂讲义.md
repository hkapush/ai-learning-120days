# Day 001 课堂讲义：搭建学习环境

## 今天这节课的意义

你要成为 AI 研发工程师，第一天不是急着学大模型，而是把“工作台”搭好。

AI 研发每天都会做这些事：

- 写 Python 代码处理数据。
- 用虚拟环境管理依赖。
- 用 Git 保存实验版本。
- 在终端运行脚本、看报错、查文件。
- 把每天的代码、结果和复盘沉淀下来。

所以今天的目标不是“安装一堆软件”，而是让你拥有一个可以持续 120 天学习和做项目的研发环境。

## 本节课目标

上完今天这节课，你应该能做到：

1. 知道 Python、虚拟环境、Git、VS Code 分别负责什么。
2. 能在终端确认这些工具是否可用。
3. 建立自己的学习仓库和每日记录方式。
4. 写出一个最小 Python 程序并运行成功。

## 老师先讲：这些工具分别是什么

### Python

Python 是你写 AI 代码的主语言。数据处理、模型训练、RAG、Agent、部署，大多数环节都离不开它。

你今天只要先记住一句话：

> Python 是你和数据、模型、实验系统沟通的主要语言。

### 虚拟环境

虚拟环境是“项目专属工具箱”。

不同项目可能需要不同版本的库，例如：

- 项目 A 需要 `torch==2.2`
- 项目 B 需要 `torch==2.5`

如果都装到系统 Python 里，迟早会冲突。虚拟环境就是为了让每个项目有自己的依赖空间。

### Git

Git 是版本管理工具。它能帮你回答：

- 我今天改了什么？
- 哪次实验效果最好？
- 代码坏了能不能回到之前版本？

AI 研发里，实验很多，Git 是你的安全绳。

### VS Code

VS Code 是代码编辑器。你用它写代码、看文件、运行终端、装插件。

## 跟做环节 1：检查工具

打开终端，在 Day-001 的 `code/` 文件夹里执行或记录这些命令：

```bash
python --version
python3 --version
git --version
pwd
ls
```

你需要观察：

- Python 是否能显示版本。
- Git 是否能显示版本。
- `pwd` 显示你当前在哪个目录。
- `ls` 显示当前目录里有什么文件。

如果有报错，把完整报错发给我，我会带你修。

## 跟做环节 2：写第一个 Python 程序

在 `code/` 文件夹里创建 `hello_ai.py`，写入：

```python
name = "AI研发工程师"
days = 120

print(f"我的目标是成为一名{name}")
print(f"我会连续学习 {days} 天")
```

然后运行：

```bash
python hello_ai.py
```

如果你的电脑需要使用 `python3`，就运行：

```bash
python3 hello_ai.py
```
