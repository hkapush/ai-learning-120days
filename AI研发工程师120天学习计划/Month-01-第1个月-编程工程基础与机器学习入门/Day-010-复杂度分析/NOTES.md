## 核心笔记

- 复杂度分析的目标不是为了考试，而是为了判断代码在数据量变大时是否还能跑得动。
- 时间复杂度关注运行步骤随输入规模 `n` 的增长情况。
- 空间复杂度关注额外占用的内存随 `n` 的增长情况。
- 一层普通循环通常先看 `O(n)`。
- 两层嵌套循环通常先看 `O(n^2)`。
- 每次翻倍或减半的循环通常先看 `O(log n)`。
- 如果额外创建了一个长度随 `n` 增长的数组、列表或哈希表，空间复杂度通常是 `O(n)`。
- 如果只用了少量固定变量，比如 `total`、`i`，空间复杂度通常是 `O(1)`。

## 代码/实验结果

- 课堂判断结果：

```text
A:
for i in range(n):
    print(i)
时间复杂度 = O(n)
```

```text
B:
for i in range(n):
    for j in range(n):
        print(i, j)
时间复杂度 = O(n^2)
```

```text
C:
i = 1
while i < n:
    i = i * 2
时间复杂度 = O(log n)
```

```text
D:
arr = []
for i in range(n):
    arr.append(i)
时间复杂度 = O(n)
空间复杂度 = O(n)
```

```text
E:
total = 0
for i in range(n):
    total += i
时间复杂度 = O(n)
空间复杂度 = O(1)
```

```text
F:
result = []
for i in range(n):
    result.append(i * 2)
时间复杂度 = O(n)
空间复杂度 = O(n)
```
